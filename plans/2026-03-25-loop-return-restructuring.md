# Loop Return Restructuring (B2b)

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Extend `go-cfg-to-structured` to handle early returns inside loops.
Returns inside `for`/`range` bodies are rewritten as control-variable assignments +
`break`, with guard-if-return statements after the loop. Combined with the existing
Case 1 (linear guard folding), the primitive produces single-exit blocks in one call.

**Design decisions (from Q&A):**
- **Type inference:** Not needed. Original return statements are preserved and moved
  to guard-ifs after the loop. No `_r0` variable — avoids AST-level type inference
  entirely. (Diverges from design doc's `_r0`/`_done` approach.)
- **Nested loops:** Handled. Bottom-up processing: inner loops are rewritten first,
  then outer loops. `_done` propagates via `if _done { break }` after inner loops.
- **Single pass:** Both Case 1 and Case 2 in one call. Case 2 runs first (loop
  rewriting), then Case 1 (guard folding). Guard-ifs generated by Case 2 are
  natural inputs to Case 1.

**Algorithm:**

```
Input:  for _, v := range items { if v.bad { return errBad } }
        return nil

Phase 1 — rewriteLoopReturns:
        var _ctl0 int
        for _, v := range items { if v.bad { _ctl0 = 1; break } }
        if _ctl0 == 1 { return errBad }
        return nil

Phase 2 — restructureStmts (existing Case 1):
        var _ctl0 int
        for _, v := range items { if v.bad { _ctl0 = 1; break } }
        if _ctl0 == 1 {
            return errBad
        } else {
            return nil
        }
```

Each loop gets a unique `_ctl<N>` int variable. Within one loop, each distinct
return site gets a numbered value (1, 2, ...). Value 0 means "loop completed
normally."

**Scope:**
- Top-level `for`/`range` with returns
- Nested `for`/`range` within loop bodies (bottom-up, recurses into ifs/blocks
  to find inner loops)
- Multiple return sites within one loop (numbered `_ctl` values)
- Switch/select containing returns within a loop → `#f` (can't replace `break`
  without labeled statements; deferred to B3)

**Not handled (deferred):**
- Loops inside ifs at the function body level (top-level only)
- Returns inside switch/select within loops (needs labeled break — B3)
- Multiple return values (B3)

**Step 3: Implement loop rewriting**

New functions in `prim_restructure.go`:

```
hasLoopReturns(stmts) bool
  — Top-level for/range whose body contains returns.

rewriteLoopReturns(stmts, counter *int) ([]ast.Stmt, bool)
  — For each top-level for/range with returns:
    1. Recursively process loop body (inner loops via ifs/blocks)
    2. Replace returns with _ctl = N; break
    3. Emit: var decl, rewritten loop, guard-ifs
  — Returns (nil, false) if any return is inside switch/select.

replaceReturnsInStmts(stmts, ctlName, retIdx *int, returns *[]*ast.ReturnStmt) ([]ast.Stmt, bool)
  — Recursive walk replacing return-stmts with ctl assignment + break.
  — Recurses into: IfStmt (body + else), BlockStmt.
  — Skips: FuncLit, ForStmt, RangeStmt (already processed).
  — Bails (false): SwitchStmt, TypeSwitchStmt, SelectStmt with returns.

AST constructors (unexported):
  makeVarDeclInt(name)       → var _ctl0 int
  makeIntAssign(name, val)   → _ctl0 = 1
  makeBreakStmt()            → break
  makeCtlGuard(name, val, ret) → if _ctl0 == 1 { return <expr> }
```

Update `PrimGoCFGToStructured` flow:
```
1. containsGoto → #f
2. rewriteLoopReturns (Case 2)  → may add guard-ifs
3. hasEarlyReturns → unchanged if none
4. restructureStmts (Case 1)    → fold guards into if/else
```
